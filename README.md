## 项目介绍

本项目为一个银行知识库管理系统（后台管理系统），用于实现知识内容的创建、审批、发布及权限化访问控制。

系统围绕“知识全生命周期管理”构建，支持从知识创建、编辑、提交审批，到审批通过后的发布与查询，并结合角色权限控制，实现不同用户对知识内容的差异化访问与操作。

## 核心功能包括：

知识管理：支持知识的创建、编辑、版本管理与发布
审批流程：支持知识提交审批、审批通过/驳回及审批记录追踪
权限控制：基于角色的访问控制（RBAC），实现页面与操作级权限管理
审计能力：记录关键操作行为，支持审计与追溯
技术架构

## 本项目采用 AI驱动的前后端开发模式，通过结构化 Prompt 设计与工具链协同，实现从需求到代码的自动化生成。

前端技术栈

- Vue3
- TypeScript
- Vite
- Element Plus
- Pinia（状态管理）

后端技术栈

- Node.js
- MongoDB

AI开发流程（核心亮点）

## 项目采用 AI 工程化开发流程：

基于需求文档（PRD）进行结构化分析
使用 ChatGPT 进行：
信息架构（IA）设计
页面级 PRD 拆解
组件与状态设计
通过 Cursor + Codex：
自动生成前后端代码骨架
分模块实现具体功能
人工进行关键节点确认（需求与结构），保证输出可控

## 项目特点

AI驱动开发：从需求到代码实现形成完整闭环
前端工程化设计：基于 Vue3 + TypeScript 构建中型后台架构
结构化 Prompt 体系：将复杂需求拆解为多阶段 Prompt 链，提高生成稳定性
权限与审批结合：适用于金融类系统的合规场景
可扩展性强：支持通过参数化 Prompt 生成不同类型后台系统
