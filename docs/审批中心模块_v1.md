# 需求变更记录：审批中心模块

## 本次目标

基于 `docs/deep-research-report.md` 与当前已实现的审批流接口，新增“审批中心”模块，支持：

- 待我审批（审核员视角）
- 我发起的（提交人视角）

## 后端变更

文件：`backend/src/routes/articles.js`

新增接口：

1. `POST /api/articles/approval/pending`
   - 权限：`approval:approve`
   - 返回当前处于 `submitted` 的版本列表（聚合文章信息）

2. `POST /api/articles/approval/initiated`
   - 权限：`kb:submit`
   - 返回当前用户提交过的版本列表（按提交时间倒序）

两类接口均支持分页参数：`page`、`pageSize`，并写入审计日志。

## 前端变更

### API

文件：`frontend/src/api/articles.ts`

新增方法：

- `fetchApprovalPending`
- `fetchApprovalInitiated`

### 类型

文件：`frontend/src/types/api.ts`

新增类型：

- `ApprovalListItem`
- `ApprovalListData`

### 页面

文件：`frontend/src/views/approval/ApprovalCenterView.vue`

功能：

- 使用 `commonPages` 渲染审批中心页面
- 顶部分段切换：
  - 待我审批
  - 我发起的
- 列表点击可进入知识详情页进行审批动作

### 路由与导航

文件：`frontend/src/router/index.ts`

- 新增路由：`/approval-center`（name: `approval-center`）
- 路由权限：`kb:read`（页面内按审批/发起权限控制可见 Tab）

文件：`frontend/src/layouts/MainLayout.vue`

- 侧边栏新增“审批中心”菜单（具备 `kb:submit` 或 `approval:approve` 权限时显示）

## 验证结果

- 后端语法检查通过
- 前端 `npm run build` 通过

## 备注

此模块为“审批中心入口与列表”能力；审批操作仍在知识详情页执行（提交/通过/驳回/发布）。
