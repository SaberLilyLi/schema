# 需求变更记录：中文乱码修复

## 问题现象

页面中文出现方块/问号乱码，主要集中在表单与正文输入区域。

## 原因分析

代码文本本身未发现坏编码字符，问题由字体链不完整导致：当前字体回退链在部分环境下未命中可显示中文的字体。

## 修复方案

### 1) 全局字体链增加中文优先字体并覆盖 Element Plus 字体变量

文件：`frontend/src/styles/global.scss`

- 新增 `--kb-font-family`（中文优先）
- 新增 `--kb-mono-font-family`（等宽场景兼容中文）
- 设置 `--el-font-family: var(--kb-font-family)`
- 为 `code` / `pre` / `textarea` 统一指定 `--kb-mono-font-family`

### 2) 编辑器文本区域改为统一等宽字体变量

文件：`frontend/src/views/knowledge/ArticleEditorView.vue`

- `.editor__textarea` 字体改为 `var(--kb-mono-font-family)`

## 验证结果

- 前端构建通过
- 中文显示恢复正常（含表单、菜单、正文输入）
