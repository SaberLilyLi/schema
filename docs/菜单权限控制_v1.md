# 需求变更记录：菜单权限控制

## 本次目标

实现左侧菜单栏权限控制：不同权限角色展示不同菜单，并将规则同步到权限文档，形成后续可维护规范。

## 前端变更

文件：`frontend/src/layouts/MainLayout.vue`

变更点：

- 将菜单改为统一配置驱动（`menuItems`）
- 每个菜单项支持权限规则：
  - `requiredPermissions`
  - `permissionMode`（`all` / `any`）
- 通过 `visibleMenuItems` 统一过滤后渲染，避免分散 `v-if`

当前菜单权限规则：

- 首页：默认可见
- 知识库：`kb:read`
- 新建知识：`kb:create`
- 审批中心：`kb:submit` 或 `approval:approve`
- 组件展示：默认可见

## 文档变更

文件：`docs/ROLE_ACCOUNTS.md`

新增章节：

- “左侧菜单可见性与权限映射”
- “权限变更维护约定”

## 验证结果

- 前端 `npm run build` 通过
- 菜单权限逻辑由单点配置统一管理，可随权限扩展
