# 需求变更记录：文章接口解耦修复

## 问题

统一改为 `POST` 后，出现了查询与新增共用 `POST /api/articles` 的冲突，导致新建知识保存后前端拿到的数据结构异常，进而触发路由参数缺失（`Missing required param "id"`）。

## 修复目标

保持接口统一 `POST`，同时将“查询 / 详情 / 新增”拆分为独立路径，避免同一路径承担不同职责。

## 后端修复

文件：`backend/src/routes/articles.js`

- 列表查询：`POST /api/articles/query`
- 详情查询：`POST /api/articles/detail/:id`
- 新增知识：`POST /api/articles/create`

## 前端修复

文件：`frontend/src/api/articles.ts`

- `fetchArticleList` -> `/articles/query`
- `fetchArticleDetail` -> `/articles/detail/:id`
- `createArticle` -> `/articles/create`

## 文档同步

文件：`docs/API.md`

- 文章列表、详情、新增接口路径同步更新为上述新路径。

## 验证结果

- 新建知识后可正确拿到 `id` 并跳转详情
- 前端构建通过
- 后端语法检查通过
