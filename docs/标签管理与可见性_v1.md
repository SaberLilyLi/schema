# 标签管理与可见性_v1

## 1. 目标

- 新增独立标签管理能力：支持新增/修改标签。
- 每个标签支持配置：颜色、含义、可见角色。
- 文章详情/列表/编辑页中的标签展示遵循“角色可见性”。

## 2. 本次变更

### 2.1 后端

- 扩展 `Tag` 模型字段：
  - `meaning`：标签含义说明
  - `visibleRoleCodes`：可见角色编码列表（空数组表示全部角色可见）
- 新增标签接口（统一 POST）：
  - `POST /api/tags/query`：标签分页查询（`kb:edit`）
  - `POST /api/tags/create`：新建标签（`kb:edit`）
  - `POST /api/tags/:id/update`：更新标签（`kb:edit`）
  - `POST /api/tags/roles`：角色列表（`kb:edit`）
  - `POST /api/tags/options`：当前用户可见标签选项（登录即可）
- 在文章列表、文章详情、可编辑数据接口中应用标签可见性过滤。

### 2.2 前端

- 新增页面：`/tag-manage`（标签管理）
  - 搜索、分页、创建、编辑
  - 可配置标签颜色、含义、可见角色
- 左侧菜单新增“标签管理”，权限：`kb:edit`。
- 新建/编辑知识页面标签输入改为“可见标签选项选择”，来源 `POST /api/tags/options`。

## 3. 权限规则

- 标签管理页与管理接口：`kb:edit`
- 标签选项查询：仅登录要求（用于业务人员选标签）
- 标签显示规则：
  - 标签未配置可见角色 -> 所有角色可见
  - 标签配置了可见角色 -> 仅命中角色用户可见

## 4. 兼容性说明

- 历史标签数据不含新字段时，默认：
  - `meaning = ""`
  - `visibleRoleCodes = []`（即全部可见）
- 不影响现有文章和标签关联关系。
