# 需求变更记录：文档接口方法统一

## 本次目标

将已产出的设计/变更文档中的接口方法描述与当前实现保持一致，统一为 `POST`。

## 本次文档修订

  - 查询参数章节标题同步改为 `POST /api/audit/logs`

- `docs/审批中心模块_v1.md`
  - `GET /api/articles/approval/pending` -> `POST /api/articles/approval/pending`
  - `GET /api/articles/approval/initiated` -> `POST /api/articles/approval/initiated`

- `docs/审计中心模块_v2.md`
  - 接口清单与“文档同步”章节中全部审计接口方法统一改为 `POST`

## 结果

设计文档、变更文档与当前代码实现一致，避免评审和联调时出现接口方法歧义。
