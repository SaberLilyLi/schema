# 需求变更记录：标签功能补齐

## 问题

“新建知识”页面缺少标签输入能力，导致文章无法在创建时直接配置标签。

## 修复内容

### 后端

文件：`backend/src/routes/articles.js`

- 引入 `Tag` 模型，新增标签写入辅助方法 `upsertArticleTags`
- 在 `POST /api/articles/create` 中支持 `tags` 入参并落库
- 在 `POST /api/articles/:id/save` 中支持 `tags` 更新
- 在 `POST /api/articles/:id/editable` 回传当前 `tags`，用于编辑页回填

### 前端

文件：`frontend/src/views/knowledge/ArticleEditorPage.vue`

- 表单新增“标签”字段
- 使用 `el-select` 多选 + 可创建模式，支持快速录入多个标签
- 新建与编辑保存时同时提交 `tags`

文件：`frontend/src/api/articles.ts`

- `createArticle`、`saveArticleEdit` 增加 `tags` 参数

文件：`frontend/src/types/api.ts`

- `ArticleEditableData` 增加 `tags: string[]`

## 验证结果

- 前端构建通过
- 新建知识可直接录入标签，详情/列表可正确显示标签
