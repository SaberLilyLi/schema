# 需求变更记录：审计中心模块

## 本次目标

基于 `docs/审计中心模块_v1.md` 落地“审计中心”能力，提供审计日志的查询、分页、详情查看，并加入菜单与权限控制。

## 后端变更

### 新增路由

文件：`backend/src/routes/audit.js`

新增接口：

1. `POST /api/audit/logs`
   - 权限：`audit:read`
   - 支持分页与过滤（`actor`、`action`、`resourceType`、`requestId`、`success`、`startAt`、`endAt`）

2. `POST /api/audit/logs/:id`
   - 权限：`audit:read`
   - 返回单条审计日志详情（含 `details`、`prevHash`、`recordHash`）

### 路由挂载

文件：`backend/src/routes/index.js`

- 新增：`apiRouter.use('/audit', auditRouter)`

## 前端变更

### API 封装

文件：`frontend/src/api/audit.ts`

- `fetchAuditLogs`
- `fetchAuditLogDetail`

### 类型定义

文件：`frontend/src/types/api.ts`

- `AuditLogDetailData`

### 页面

文件：`frontend/src/views/audit/AuditCenterView.vue`

功能：

- 使用 `commonPages` + `commonSearch` + `commonTable` 展示审计日志列表
- 支持条件查询与分页
- 点击行打开详情抽屉，查看完整审计信息及哈希链字段

### 菜单与路由

文件：`frontend/src/router/index.ts`

- 新增路由：`/audit-center`
- 路由权限：`audit:read`

文件：`frontend/src/layouts/MainLayout.vue`

- 菜单新增“审计中心”
- 菜单权限：`audit:read`
