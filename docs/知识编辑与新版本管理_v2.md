# 需求变更记录：知识编辑与新版本管理

## 本次目标

落地“已有知识可编辑 + 已发布可新开版本草稿”的能力，形成完整版本演进闭环。

## 后端变更

文件：`backend/src/routes/articles.js`

新增接口：

1. `POST /api/articles/:id/editable`
   - 权限：`kb:edit`
   - 返回当前知识可编辑数据与编辑策略（`update_current` / `new_version` / `readonly`）

2. `POST /api/articles/:id/save`
   - 权限：`kb:edit`
   - 按当前流程状态保存：
     - `draft/rejected`：覆盖当前版本
     - `published`：创建新版本草稿并切换 `currentVersionId`
     - `submitted/approved`：拒绝保存

补充逻辑：

- 驳回时已记录审批人和审批时间（前一模块已接入），可与版本历史联动展示
- 新增审计动作：`kb.edit.read`、`kb.edit.save`、`kb.version.create`
