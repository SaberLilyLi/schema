# 信息架构(IA)设计文档

## 一、项目说明

本项目为一个**金融后台管理系统(知识库方向)**

系统具备以下核心能力：

- 知识库全生命周期(新建-审核-发布-修改)
- 多角色权限控制(管理员、审核员、查阅员)
- 审批流程支持
- 审计日志与追踪操作

---

## 二、业务划分

1.首页2.知识库管理(核心) 3.新建/编辑知识库4.版本历史5.标签管理

## 三、页面结构

1. 首页

- 仪表盘

2. 知识库管理（核心域）

- 知识列表（核心）
- 知识详情
- 新建/编辑知识
- 版本历史
- 标签管理

3. 审批中心

- 待审批列表
- 已审批记录
- 审批详情

4. 审计中心

- 操作日志
- 登录日志
- 审计查询

5. 系统管理

- 用户管理
- 角色管理
- 权限配置

## 页面类型映射

## 核心流程

创建知识 → 编辑 → 提交审批 → 审批 → 发布 → 查看 → 修改 → 再审批

## 六、页面说明（核心页面）

1. 知识列表（核心页面）
   类型：列表页
   角色：所有用户
   功能：
   搜索 / 筛选
   分页
   查看详情
   编辑（有权限）
   删除（有权限）
   提交审批
   状态：
   草稿 / 审批中 / 已发布 / 已驳回
2. 知识详情
   类型：详情页
   功能：
   查看知识内容
   查看版本历史
   查看审批记录
   操作入口（编辑 / 审批）
3. 新建 / 编辑知识
   类型：编辑页
   功能：
   表单填写
   富文本编辑
   自动保存草稿
   提交审批
   状态：
   草稿
   驳回后修改
4. 审批详情
   类型：审批页
   功能：
   查看知识内容
   审批（通过 / 驳回）
   填写审批意见
   查看审批历史

## 七、权限结构

1. 角色划分
   管理员
   审核员
   查阅员

2. 页面权限（菜单可见性）
   | 页面 | 管理员 | 审核员 | 查阅员 |
   | ----- | --- | --- | --- |
   | 首页 | ✅ | ✅ | ✅ |
   | 知识库管理 | ✅ | ✅ | ✅ |
   | 审批中心 | ✅ | ✅ | ❌ |
   | 审计中心 | ✅ | ❌ | ❌ |
   | 系统管理 | ✅ | ❌ | ❌ |

3. 操作权限（按钮级）
   管理员
   创建 / 编辑 / 删除知识
   提交审批
   审批通过 / 驳回
   发布 / 撤回
   用户与权限管理
   审核员
   审批（通过 / 驳回）
   查看知识
   不可删除已发布内容
   查阅员
   查看知识
   编辑自己创建的知识（被驳回状态）
   不可审批
4. 权限设计说明
   前端只负责：
   页面可见性控制
   按钮级显示控制
   后端负责：
   最终权限校验
   权限分层：
   页面权限（路由级）
   操作权限（组件级）

## 八、设计原则

1. 业务域优先（菜单设计以业务为中心）
2. 页面职责单一（避免一个页面承载过多功能）
3. 权限前后端分离（前端展示，后端校验）
4. 页面类型标准化（支持模板复用）
5. 主链路优先（优先实现核心业务流程）
