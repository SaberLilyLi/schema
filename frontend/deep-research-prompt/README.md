# Deep Research Prompt（前端 Prompt 工程化实践）

## 一、项目背景

在使用 AI（如 ChatGPT / Codex / Cursor）进行前端开发时，
常见问题包括：

- 输出不稳定（每次结果不一致）
- 生成代码不可维护
- 无法从需求系统化推导前端结构
- prompt 分散，缺乏复用能力

本项目基于一个“银行知识库系统”需求说明，
将其反向拆解为一套 **结构化 Prompt 工程体系**，
用于提升 AI 在前端开发中的稳定性、可控性与工程化能力。

---

## 二、项目目标

本项目不直接生成业务系统，而是构建：

> ✅ 一套“前端 Prompt Pipeline（提示词执行链）”

实现：

1. 将复杂需求拆解为多个可控 Prompt
2. 让 AI 按步骤生成前端设计与代码
3. 提高输出一致性与可复用性
4. 支持参数化扩展到其他后台系统

---

## 三、核心能力

本项目聚焦以下能力：

### 1. Prompt 链路设计（Prompt Chain）

将需求拆分为多个阶段，而不是一次性生成结果：

- 项目范围收敛
- 信息架构设计
- 页面级 PRD
- 组件拆分
- 状态与接口设计
- 代码骨架生成
- 代码审查与优化

---

### 2. 前端工程化思维

- Vue3 + TypeScript + Vite
- 组件化设计（Container / Business / Base）
- 状态管理（Pinia）
- API 分层
- 权限展示（前端层）

---

### 3. Prompt 模板化（Templates）

支持将 prompt 抽象为模板，例如：

- 列表页模板
- 详情页模板
- 表单页模板
- 审批流页面模板
- 权限管理页面模板

---

### 4. 参数化能力（可扩展）

通过变量驱动不同系统生成：

```md
系统类型：{{system_type}}
行业：{{industry}}
核心模块：{{modules}}
权限模型：{{permission_mode}}
```
